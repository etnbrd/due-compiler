Dues make your callback behave well

If you use to write asynchronous Javascript code, you might have encounter the Callback hell aka Pyramid of Doom. To avoid this pattern, you need to cleanly arrange your asynchronous execution flow, for example with Promises. A Promise is a cleaner pattern allowing to write asynchronous sequence as a cascade of call, instead of a pyramide of call. Promises are coming in ECMAScript 6.

It is time to switch, and start writing better code.

However, you might already have tons of lines with callbacks. and you don't want to refactor everything. As a first step toward Promises, here are two Callbacks to Due compiler. single file compiler project compiler

Dues

Dues are like Promises, they provide the chainable 
then
method. However, they don't expect two callbacks like promises, but only one. To use the compiler be aware that a Due, like a Promise, can only replace continuations : callbacks called only once, and asynchronously.
